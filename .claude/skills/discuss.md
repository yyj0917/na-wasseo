# Intent Alignment (의도 정렬)

## 목적

코드를 한 줄도 쓰기 전에, 유저의 의도를 정확히 이해했는지 확인.

## 반드시 확인할 5가지

| #   | 질문                         | 예시 (나 왔어! 맥락)                            |
| --- | ---------------------------- | ----------------------------------------------- |
| 1   | **무엇을** 만드는가?         | "출퇴근 버튼" = 출석만? 퇴근도? 상태 전이 포함? |
| 2   | **누가** 쓰는가?             | 팀원? 어드민? 비로그인?                         |
| 3   | **어디에** 들어가는가?       | /team 페이지? /admin? 새 페이지?                |
| 4   | **기존 코드**와 어떻게 연결? | Firestore 기존 컬렉션? 새 컬렉션?               |
| 5   | **완료 기준**은?             | 어떤 상태가 되면 "됐다"인가?                    |

## 수행 절차

1. 유저 요청을 내 말로 다시 정리
2. 위 5가지 중 모호한 것을 질문
3. 유저 확인 시 요약 블록 출력:

```
## ✅ 의도 정렬 완료
- **만들 것**: {구체적 설명}
- **대상 유저**: {누가 사용}
- **위치**: {어떤 라우트/컴포넌트}
- **DB 변경**: {Firestore 컬렉션 변경 사항}
- **완료 기준**: {어떻게 되면 완성}
```

4. 확인 후 → `/plan`으로 안내

## 절대 하지 않을 것

- 코드 작성
- 파일 생성
- 구현 방법 제안 (/plan의 역할)
- 유저가 "그냥 만들어"라고 해도 최소 1번은 의도 확인
