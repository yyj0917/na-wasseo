name: discuss
description: 구현 전 의도 정렬. 모호한 요청을 명확하게 만든다. 코드를 절대 작성하지 않는다.
---

유저의 요청을 분석하고 다음을 수행:

1. 요청을 내 말로 다시 정리
2. 아래 표의 빈칸을 유저와 함께 채우기:

| 항목           | 확인 내용                     |
| -------------- | ----------------------------- |
| 만들 것        | (구체적으로)                  |
| 대상 유저      | (팀원? 어드민?)               |
| 위치           | (어떤 라우트, 기존/신규)      |
| DB 변경        | (Firestore 컬렉션 추가/수정?) |
| 기존 코드 영향 | (어떤 파일이 변경?)           |
| 완료 기준      | (어떻게 되면 "됐다"?)         |

3. 모든 항목 채워지면 요약 블록 출력
4. 유저 확인 후 → `/plan`으로 전환 안내

**절대 하지 않을 것**: 코드 작성, 파일 생성, 구현 방법 제안
